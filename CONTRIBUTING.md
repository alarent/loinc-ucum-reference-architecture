# Contributing
Спасибо за интерес к этой reference-архитектуре.

## Что это
Это архитектурный артефакт (design v1.0), а не библиотека. Реализации, которую можно собрать или протестировать, пока нет — она планируется в Phase 2. Поэтому вклад — это вклад в дизайн: ADR, worked examples, схемы, граничные случаи.

## Как внести вклад
- Открой GitHub Issue, если нашёл: граничный случай, который контракт не покрывает; рассогласование между документами, схемами и примерами; предложение по изменению архитектуры.
- Архитектурные изменения оцениваются по [scope.md](docs/scope.md); если они затрагивают инвариант — оформляются как новый ADR или supersede существующего (см. [ADR-индекс](docs/adr/README.md)).
- Изменения выходного контракта означают major version bump (см. [versioning.md](docs/versioning.md)).

## Конвенции
- Проза в docs — под CC-BY-4.0; код, схемы, примеры — под Apache-2.0. Граница по расширению файла: .md — документация; .json и код — Apache.
- Указывай номера ADR и examples, которых касается изменение.
- Язык: русская проза, английский только в стандартных терминах.

## Вне scope для вклада в v1.0
- Reference implementation, адаптеры форматов, production-grade данные LOINC/UCUM — это Phase 2 (см. [scope.md](docs/scope.md)).
